# План: Полное описание всех графических ассетов PVOGame

## Контекст
Все визуальные элементы игры сейчас — цветные квадраты и программно-сгенерированные фигуры. Нужно создать описания для генерации изображений каждого элемента.

**Стиль UI/интерфейса**: SAM Simulator — военная консоль ПВО, CRT-мониторы с зелёным фосфором, радарные развёртки, тёмный фон с зелёным/янтарным текстом, металлические панели, советская эстетика систем ПВО.

**Стиль игровой графики**: Defense Zone 3 — детализированная военная графика, реалистичное оружие/техника/ракеты, реалистичный дым и огонь, камуфляж, вид сверху.

**Технические требования**: 2D спрайты, прозрачный фон (PNG), рендер на @3x (×3 от pt-размера), затем даунскейл для @2x.

## КАТЕГОРИЯ 1: БАШНИ ПВО (8 типов, многослойные спрайты)

Каждая башня состоит из **нескольких отдельных спрайтов-слоёв**: база (статичная) + турель/оружие (вращается или анимируется) + вспышка выстрела (для пушечных). Все слои — вид сверху, прозрачный фон PNG. Рендер базы: **128×128px**. Размеры турелей и вспышек указаны индивидуально.

**Важно для художника**: Турель рисуется ОТДЕЛЬНО от базы. Оружие/стволы направлены **строго вверх (на 12 часов)** — это дефолтное положение, поворот делается кодом. Центр вращения турели = центр спрайта (если не указано иное через anchor point).

## Архитектура многослойных спрайтов башен

### Иерархия нод в SpriteKit

Каждая башня — это контейнер `SKSpriteNode(color: .clear, size: 28×28pt)`, внутри которого слои:

```
containerNode (SKSpriteNode clear) — worldPosition, zPosition=25, physicsBody
  ├── baseSprite (SKSpriteNode)    — zPos=0 (relative), НИКОГДА не вращается
  ├── turretSprite (SKSpriteNode)  — zPos=1, вращается (tracking) / анимируется (прочие)
  │     └── muzzleFlash (SKSpriteNode) — zPos=2, child турели, alpha=0 по умолчанию
  └── ammoDots, reloadArc, smoke   — без изменений, children containerNode
```

`SpriteComponent.spriteNode` = containerNode (прозрачный 28×28). Вся существующая логика позиционирования, физики и child-нод (точки боезапаса, дуга перезарядки, дым повреждения) продолжает работать без изменений.

### Таблица анимаций

| Тип анимации | Метод SpriteKit | Когда | Башни |
|-------------|----------------|-------|-------|
| Вращение к цели | `turretNode.zRotation` per-frame (3π рад/с) | Есть цель | ЗУ, ЗРПК, Гепард |
| Возврат в 0° | `turretNode.zRotation` per-frame (1.5π рад/с) | Нет цели | ЗУ, ЗРПК, Гепард |
| Вращение радара 360° | `SKAction.repeatForever(rotate(byAngle: 2π, duration: 3.0))` | Всегда (пауза при disabled) | РЛС |
| Вспышка выстрела | `muzzleFlash.alpha = 1.0` → `fadeOut(0.02-0.05с)` | Каждый выстрел | ЗУ, ЗРПК, Гепард |
| Отдача ПУ (ракетной) | `SKAction.scaleY(to: 0.93, 0.08с)` → `scaleY(1.0, 0.15с)` | Пуск ракеты | С-300, Patriot |
| Отдача ПЗРК | `SKAction.moveBy(x:0, y:-2, 0.06с)` → `moveBy(0, +2, 0.1с)` | Пуск ракеты | ПЗРК |
| Пульсация РЭБ | `SKAction.scale(1.0→1.05→1.0, 2с)` repeatForever | Всегда (активное глушение) | РЭБ |

### Точка вылета снаряда

Для **пушечных башен** (ЗУ, ЗРПК, Гепард) начало трассера = кончик стволов:
```
barrelTip = containerNode.position + rotate(muzzleOffset, by: turretNode.zRotation)
```
Где `muzzleOffset` — смещение от центра турели до дульного среза в локальных координатах (например, `(0, +15pt)` для ЗУ).

Для **ракетных башен** (С-300, Patriot, ПЗРК) начало ракеты = центр контейнера (без изменений).

1. **Башни (многослойные)** — переделать `TowerEntity.init()`: создать containerNode (clear SKSpriteNode), добавить baseSprite + turretSprite как children. Добавить свойства `turretNode`, `muzzleFlashNode` в TowerEntity

5. **TowerEntity disabled/repair** — применять colorBlendFactor к baseSprite и turretSprite отдельно
6. **Дроны** — заменить единый "Drone" и colorBlendFactor на уникальные спрайты по типам
7. **Тайлы** — заменить `colorForTerrain()` в `InPlaySKScene.swift` на текстурные тайлы (7 типов: ground, highGround, blocked, headquarters, settlement, concealed, valley)
